# CallbackScheduler — учебное задание

## Цель

Реализовать класс `CallbackScheduler`, который позволяет запланировать вызов
колбека (`std::function<void()>`) в заданный момент времени.

## Интерфейс

```cpp
using TimePoint = std::chrono::time_point<std::chrono::system_clock>;

class CallbackScheduler
{
public:
    using TaskId = std::uint64_t;

    TaskId Schedule(std::function<void()> callback, TimePoint when);
    bool Cancel(TaskId id);
};
```

## Требования к реализации

1. Метод `Schedule` должен принимать задачу и время запуска и возвращать
   идентификатор задачи.
2. Колбек должен выполняться **не раньше** времени `when`.
3. Нужно поддержать планирование нескольких колбеков.
4. Если колбек запланирован на время в прошлом, его нужно выполнить как можно
   скорее.
5. Нужно реализовать `Cancel(TaskId id)` для отмены запланированной задачи.
6. Отменённая задача не должна быть выполнена.
7. Решение должно быть потокобезопасным.
8. Исключения, брошенные внутри пользовательского колбека, не должны падать за
   пределы рабочего потока планировщика.

## Что можно использовать

- `std::thread`
- `std::mutex`
- `std::condition_variable`
- `std::priority_queue` / `std::multimap` / другие структуры данных
- Любые средства стандартной библиотеки C++20

## Критерии проверки

- Корректность по времени выполнения колбеков
- Корректная отмена запланированных задач
- Отсутствие гонок данных и дедлоков
- Стабильная работа при нескольких запланированных задачах
- Читаемость и аккуратность кода
